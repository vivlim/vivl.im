A Front Matter

2024-05-18
:: tags:  #meta

I was initially using zola but then learned that it doesn't support top-level tags in front matter. zola expects tags to be within taxonomies

---
title: Okay it's website again
date: 2024-05-17
taxonomies:
  tags:
  - meta
---

if I put tags under taxonomies it's OK.

but I want to draft posts in Obsidian, and that expects tags to be at the top level, and gives me some nice ui for setting them:

![[Pasted image 20240519004719.png]]

that ui represents this:

---
tags:
  - meta
title: Okay it's website again
date: 2024-05-17
---

one option i could have taken is to patch the markdown files to move tags into taxonomies at build time, but that would mean I wouldn't be able to use zola serve & see changes locally that involved tags. that's the direction this thread I found went in.

another option is to switch away from zola to a different static site generator, but i wasn't feeling that, i'd rather work with zola since i have more experience with rust and tera.

so instead i forked zola and hacked in top-level tags for myself to use.
